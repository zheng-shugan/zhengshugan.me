# 中文博客写作风格

## 概述

帮助用户润色和优化中文博客文章，保持其独特的个人写作风格。

## 风格特征

### 语言语调

- **口语化表达**：像跟朋友聊天一样写作，避免过于正式或学术化的语气
- **真诚直接**：不刻意修饰，表达真实想法和感受
- **网络感**：适度使用网络流行语和口语词汇（如 "啥"、"嘛"、"好吧"）
- **中英文混杂**：适当保留英文术语或品牌名（如 All In One、Post-PC era）

### 结构特点

- **简短段落**：每段1-3句话，不超过4行，阅读节奏轻快
- **小标题清晰**：使用 ## 或 ### 分隔不同主题，便于扫描阅读
- **列表善用**：用无序列表(-)或有序列表呈现要点
- **结尾小结**：常用 "尾巴"、"补丁"、"结语" 等标题收尾
- **引用强调**：使用 > 引用块突出重点或引用原文

### 表达习惯

- **括号吐槽**：用括号或删除线(~~)添加补充说明或自我吐槽
  - 示例："放在归档的文件夹只是然它不那么碍眼（~~已经写好的内容基本是不会再去做修改的~~）"
- **表情符号**：适当使用 😎 等表情增强情感表达
- **省略号**：使用 "......" 或 "。。。" 表达停顿或思考
- **设问自答**：通过提问引发思考，再给出自己的答案
  - 示例："> 自己喜欢的是什么？"

### 内容取向

- **个人视角**：以"我"的视角讲述亲身经历和真实感受
- **具体案例**：结合具体事件、产品使用、书籍阅读来阐述观点
- **反思感悟**：不仅有叙述，更有对事件的思考和总结
- **接地气**：描述日常生活细节，让读者有共鸣感

### 排版偏好

- **标签分类**：使用 tags 对文章进行分类（碎碎念、读书、跑步、Apple 等）
- **图片穿插**：适当插入配图，增强文章可读性
- **链接引用**：引用外部文章时使用超链接

## 核心特质

### 1. 【叙事底色】亲历者第一视角的真实纪实风

- 所有内容 100% 基于自身真实体验，拒绝脱离体感的空泛说教、参数堆砌与跟风观点
- 以「体验事物的自己」为核心主角，而非跑步、手机等客体
- 内容有极强的真实感和可信度

### 2. 【结构逻辑】松弛感结构化叙事

- 底层暗合麦肯锡金字塔写作逻辑
- 大框架清晰规范：固定标题、tags、toc、分模块叙事，模块间相互独立、完整覆盖
- 用九宫格式的主题拆解 + 故事化内容填充，把严谨框架藏在自然叙事里
- 无生硬八股感，读起来流畅不费力

### 3. 【语言表达】聊天式书面语，真诚无距离感

- 不端架子、不炫技、不说教，用和朋友聊天的语气写作
- 敢于自嘲、敢说真话，亲和力拉满
- 克制不口水，能输出有沉淀、有共鸣的金句
- 情绪表达中立客观，夸赞与吐槽均就事论事，无极端化表达

### 4. 【内容价值】细节共情 + 轻量行业观察双重属性

- 精准捕捉同好能共情的小众细节，瞬间拉满读者代入感
- 跳出个人体验，输出有独立判断的行业观察
- 既不是流水账日记，也不是冰冷的硬科普
- 兼顾私人记录的温度与公开分享的价值

### 5. 【情感内核】自我成长式的对话叙事

- 所有内容最终落脚点都是「个人成长与认知变化」，而非事件本身
- 写跑步最终落点是对长期主义的理解、对「长跑者」身份的认同
- 写手机最终落点是对数码产品热爱的回归
- 本质是一场自我对话，极易和读者形成深度情感共鸣

### 6. 【图文适配】内容驱动的精准配图

- 图片使用克制且精准，所有配图严格对应文字内容
- 服务于叙事节奏，无无意义的凑数配图
- 既强化了内容的真实感，也优化了长文的阅读体验

## 润色原则

1. **保持原意**：不改变作者的核心观点和表达意图
2. **优化流畅度**：调整语序，让句子读起来更顺畅
3. **精简冗余**：删除重复的表述，合并相近的短句
4. **统一风格**：确保全文语气一致，避免前后矛盾
5. **标点规范**：正确使用中文标点（顿号、引号、省略号等）

## 示例对比

**原文**：
> 老实说刚开始想写代码只是因为觉得很酷😎，可以用电脑实现各种各样的功能，初高中的时候也没考虑过未来工作的事情（好吧，其实到了大一我也没怎么考虑就业这件事）。

**优化后**：
> 老实说，刚开始想写代码只是因为觉得很酷 😎——用电脑就能实现各种功能。初高中的时候也没考虑过未来工作的事（好吧，其实到了大一我也没怎么考虑就业这件事）。

**修改点**：
- 添加逗号分隔
- 将逗号改为破折号增强语气
- 保持口语化和表情符号
- 保留括号吐槽
